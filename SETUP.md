# 신덕교회 홈페이지 (tbchch) iCloud 개발 환경 안내서

이 프로젝트는 **iCloud Drive** 내의 `안티그라비트/tbchch` 폴더를 단일 진실 공급원(SST)으로 사용하도록 구성되어 있습니다. 이를 통해 인터넷이 연결된 어디서나 동일한 최신 상태의 코드로 작업할 수 있습니다.

---

## 💡 iCloud 환경에서의 핵심 주의 사항 (필독)

macOS의 iCloud Drive는 하드 디스크 공간이 부족할 때 파일의 로컬 다운로드를 해제(오프로드)하여 클라우드에만 보관하고, 로컬에는 `.파일명.확장자.icloud` 형태의 임시 포인터 파일만 남겨두는 기능이 있습니다.

이로 인해 터미널이나 Git에서 소스 코드가 지워진 것처럼 보이거나("deleted" 상태로 감지됨), 프로젝트 빌드 시 `Cannot find module ...` 에러가 발생할 수 있습니다.

이를 예방하고 편리하게 작업하기 위해 **자동 복원 스크립트(`run-dev.sh`)**를 제공합니다.

---

## 🚀 개발 및 빌드 명령어

### 1. 개발 서버 구동 (추천)
터미널을 열고 프로젝트 루트 폴더에서 아래 스크립트를 실행합니다. 
스크립트가 실행될 때 **iCloud에 의해 오프로드된 소스 코드를 자동으로 검출하고 강제 다운로드(brctl)한 뒤** 서버를 안전하게 구동합니다.

```bash
./run-dev.sh
```

*(또는 일반적인 `npm run dev` 명령어도 사용 가능하지만, 파일 다운로드 누락이 의심될 때는 `./run-dev.sh`를 실행하는 것이 안전합니다.)*

### 2. 프로덕션 빌드 테스트
실제 배포하기 전 빌드가 정상적으로 완료되는지 테스트하려면 아래 명령어를 사용합니다.
```bash
npm run build
```

### 3. 빌드 결과물 로컬 확인
빌드된 파일들을 로컬 웹 서버로 미리보기 하려면 아래 명령어를 사용합니다.
```bash
npm run preview
```

## 📂 폴더 구조 및 규칙
- 기존 로컬 홈 디렉터리에 있던 `~/tbchch` 및 기타 로컬 개별 복사본 작업은 무시해 주시고, 반드시 **iCloud Drive 내의 `안티그라비트/tbchch` 폴더**를 VS Code 등의 에디터로 열어서 편집해 주세요.
- Git을 사용할 경우, 커밋 및 푸시를 하기 전에 `./run-dev.sh` 등을 통해 파일들이 모두 로컬에 내려받아졌는지 확인하시는 것을 권장합니다.
